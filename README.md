# Проверь себя
Предлагаю  MVP - отдельный раздел с вопросами по разным тематикам и инструментам. Открытый только для учеников. Без привязки к недельным планам
Следующая итерация. Назначать вопросы в недельный план. 

## Концепция 

```yaml
Проверь себя: !concept
  определение: Сервис для подготовки к собеседованию по теории Python.
  Открытая программа тестирования: !example |
    В сервисе проверь себя ученик или любой другой (неавторизованный) пользователь может отвечать на вопросы и сразу же получать результат прохождения теста.
  Закрытая программа тестирования: !extension |
    Только авторизованый пользователь с определенными правами доступа может видеть вопросы и отвечать на них.
Вопрос: !concept
  определение: Вопрос содержит: текст вопроса, варианты ответов, пояснение к ошибочному ответу, пояснение к правильному ответу, текстовый комментарий в формате html (с возможностью добавить ссылку), ссылку на тематику, ссылку на инструмент, сложность, статус (active, disable)

  С одним правильным ответом: !example |
    Вопрос переходит в статус "решен", если выбран правильный ответ.
  С множеством правильных ответов: !extension |
    Вопрос переходит в статус "решен", если выбраны все правильные ответы.

  Новый: !example |
    Статус вопроса по умолчанию, когда ученик не отвечал ни разу на вопрос.
  Решенный: !example |
    Временный статус вопроса. Вопрос переходит в статус "решен", если выбран правильный ответ. Далее вопрос переходит в статус "На повторении". 
  Ошибочный: !example |
    Вопрос переходит в статус "ошибка", если выбран не правильный ответ. Временный статус вопроса, вопрос возвращается в статус "Новый" при переходе к следующему вопросу
  На повторении: !example |
   Вопрос был решен в период больший или равно 2х+1, где х-количество правильно данных ответов подряд на этот вопрос.
  В очереди: !example |
    Вопрос назначен ученику в недельном плане

С тематикой: !example |
  Вопросы тематики доступны после перехода на следующий курс
Без тематики: !example |
  Вопросы открыты для всех пользователей (для неавторизованных в том числе)

Сложный вопрос: !example |
  Вопрос для мидл разработчика или вопрос требующий знаний по нескольким инструментам
Средний вопрос: !example |
  Вопрос для джуна или вопрос требующий знаний по не более 2м инструментам
Легкий вопрос: !example |
   Вопрос для новичка, требующий знаний по одному инструменту

Напоминание о необходимости пройти тесты: !concept
  определение: Попап отображаемый на странице профиля после авторизации. Уведомление поступает автоматически в следующих ситуациях:
  На повторении: !example |
    Более 5 вопросов перешли в статус "на повторении".
  Вопрос отправил ментор: !extension |
    Вопрос переходит в статус "решен", если выбран правильный ответ.
Тематика вопроса: !concept
  определение: Тематика вопроса = курс. Используется для объединения  и фильтрации вопросов. Тематика отображается на сайте, если имеет хотя бы 1 привязанный к ней вопрос.Тематика состоит из: название, картинка (svg или png размер 33х33), описание тематики
  Базовый Python: !example |
    -.
  Асинхронный Python: !example |
    -.
  SQL: !example |
    -.
  Структуры данных: !example |
    -.
Прогресс пользователя: !concept
  определение: прогресс пользователя должен отображать всем заинтересованым лицам общую статистику пользователя в сервисе Проверь себя. Прогресс пользователя состоит из следующих частей
  Решено из Всего вопросов доступно: !example |
    Количество вопросов в статусе "Решен"
  На повторении: !example |
    Количество вопросов в статусе "На повторении"
  Процент правильных ответов: !example |
    Расчитывается как (количество переходов в статус "ошибка" / Количество переходов с статус "Решен") * 100
  Ожидают выполнения: !extension |
    Количество вопросов в статусе "В очереди"

Недельный план по разделу Проверь себя: !concept
  определение: Индивидуальный план обучения под каждого ученика, который расчитан в соответствии с временем ученика, затраченым на обучение. На раздел Проверь себя ученик должен тратить не более часа в неделю (расчет 15 минут за 4 дня в неделю). План формируется следующим образом - сначала в план добавляются вопросы на изучение, если остается время, то добавляются вопросы на повторение.

  План на повторение: !example |
    часть недельного плана состаящая из вопросов в статусе "на повторение". Для ученика запись в плане выглядит так: "Повтори изученое ранее." Кликнув на строку переходишь на страницу Проверь себя с фильтром по вопросам в статусе "На повторении". Сортировка от старого к новому
  План на изучение: !example |  
    часть недельного плана состаящая из вопросов в статусе "В ожидании". Для ученика запись в плане выглядит так: "Ответь на вопросы из собеседований по изученным темам ("<Название инструмента 1>", "<Название инструмента 2>"). Выполено 0 из 15 ". Кликнув на строку переходишь на Проверь себя с фильтром по вопросам в статусе "В ожидании"

  Автоматически сформированный недельный плана: !extension |
    После изучения инструмента вопросы по этому инструменту автоматически добавляются в недельный план ученика, если остается время, то добавляются вопросы в статусе "на повторение". При автоматическом формировании недельного плана должен учитываться статус вопроса (если ученик заранее решил вопросы, то они попадают только в недельный план для повторения)
  Сформирован вручную: !example |
    Ментор вручную добавляет/удаляет вопросы из недельного плана. Вручную должна быть возможность выбрать любое количество вопросов, вне зависимость от временых затрат на вопрос.

  Выполненный недельный план: !example |
    Недельный план по разделу Проверь себя считается выполненым, если ученик ответил на все вопросы в статусе "В ожидании" и поставил галочку о прохождении. Вопросы в статусе "на повторении" не влияют на выполенение недельного плана.
  В ожидании выполнения: !example |
```

## User story
Вводные данные: 
Разработ и реализован дизайн пользовательской стороны сервиса. Его можно посмотреть по ссылке http://test.hmlanding.ru/dvmn/ 
Тут то что уже сделано https://gitlab.levelupdev.ru/dvmn/lms/dvmn.org/-/merge_requests/160 
Общее описание возможностей функционала в соответствии с ролью:
Возможности ученика по профессии:
  Отслеживать свой прогресс
  Отвечать на вопросы из недельного плана, полученный от ментора
  Отвечать на доступные для этого ученика вопросы (не из недельного плана)
  Фильтровать вопросы
  Повторно отвечать на вопросы в статусе "На повторении"
  Повторно отвечать на вопросы в статусе "Решено"

Возможности ментора:
  Выделять вопросы для недельного плана на изучение
  Выделять вопросы для недельного плана на повторение
  Отслеживать прогресс ученика

Возможности Главы подразделения «Подготовка к тех.интервью»:
  Оценить качество наполнения тренажера
  Фильтровать вопросы по тематике
  Фильтровать вопросы по инструментам
  Смотреть статистику по вопросам
  Скрыть/показать вопрос

Возможности администратора обучения:
  Открывать вручную доступ к тематикам и инструментам для ученика

Возможности контент-менеджера:
  Создание/ редактирование вопросов
  Создание/ редактирование справочника тематик
  Создание/ редактирование справочника инструментов
  
Автоматический функционал:
  Открытие доступа к вопросам для ученика
  Проверка верности установленой галочки о прохождении вопросов недельного плана
  Перевод вопросов в статус "На повторении"
  Формирование недельного плана
  Отправка уведомлений ученику о непройденых вопросах в недельном плане
  Отправка уведомлений Главе подразделения об инструментах не покрытых вопросами
